# Phase 2 设计：Cross-Sectional Encoder + 抗过拟合

> 日期: 2026-04-25
> 状态: 设计完成，待实施
> 前置: Phase 1 完成，Eval IC=0.058 > 0.03 阈值

## 〇、核心目标

1. **加 Cross-Sectional Encoder** — 让模型看到截面信息（"为什么选 A 不选 B"）
2. **解决严重过拟合** — Val IC=0.91 vs Eval IC=0.058，差距 0.85

---

## 一、架构改动 (`attention_learner.py`)

### 当前 Phase 1

```
Input (batch, 60, 47) → Temporal Transformer → summary token → heads
```

### Phase 2

```
Input (N_stocks, 60, 47)
       ↓
[Temporal Transformer]           ← 复用 Phase 1，per-stock 独立
       ↓ (N_stocks, d_model=128)
[Cross-Sectional Transformer]    ← 新增，2 层，stocks 互相 attend
       ↓ (N_stocks, d_model)
[Multi-Task Heads]               ← 复用，IC loss 变成天然截面内计算
```

### 新增模块 `CrossSectionalEncoder`

- 2 层 TransformerEncoder, 8 heads, d_model=128
- 输入: `(1, N_stocks, d_model)` — 每个 stock 是一个 "token"
- 可学习 stock-count 位置编码（按截面排名注入，不是固定 ID）
- 输出: `(1, N_stocks, d_model)` — 每只 stock 的表示已融合截面信息
- **参数增量**: ~265K（2 层 encoder），总参数 ~840K

### 模型前向传播伪代码

```python
class FactorAttentionModel(nn.Module):
    def forward_cross_sectional(self, x):
        """
        x: (N_stocks, seq_len, n_factors)
        """
        # 1. Temporal: 每只 stock 独立编码
        h = self.embedding(x) + self.pos_encoding          # (N, seq, d)
        summary = self.summary_token.expand(N, -1, -1)
        h = torch.cat([summary, h], dim=1)                 # (N, seq+1, d)
        h = self.temporal_transformer(h)                    # (N, seq+1, d)
        stock_embeddings = h[:, 0, :]                       # (N, d) summary tokens

        # 2. Cross-Sectional: stocks 互相 attend
        cs_input = stock_embeddings.unsqueeze(0)            # (1, N, d)
        cs_output = self.cross_sectional_transformer(cs_input)  # (1, N, d)
        cs_output = cs_output.squeeze(0)                    # (N, d)

        # 3. Residual: 截面输出 + 时序输出
        final = stock_embeddings + cs_output                # (N, d)

        # 4. Multi-task heads (同 Phase 1)
        regression = self.regression_head(final)
        classification = self.classification_head(final)
        quantiles = self._monotone_quantiles(final)
        return {...}
```

## 二、数据管道改动 (`train_attention.py`)

### 当前问题

样本是 `(stock, date)` 对，随机 shuffle，**丢失了截面结构**。

### Phase 2 数据格式

```python

# Phase 2: 按日期分组
cross_sections = {

        "y_reg": np.ndarray,   # (N_stocks_on_date,)
        "y_cls": np.ndarray,   # (N_stocks_on_date,)
        "symbols": list[str],
    }
}
```

### 变长截面处理

不同日期股票数不同（300~500），方案：
- **每个 batch = 一个交易日**，不 batch 多个日期，避免 padding
- 如果 GPU 内存允许，可以一次处理 2~3 天（padding + attention mask）

## 三、正则化大修（最关键）

Phase 1 过拟合根源：
- 120K 训练样本 vs 575K 参数（接近 1:5）
- 无数据增强
- Val 集是从 train 随机切分（同分布），Eval 是未来年份（分布偏移）

### 正则化方案

| 手段 | Phase 1 | Phase 2 |
|------|---------|---------|
| Dropout | 0.1 | **0.2** |
| Weight decay | 1e-4 | **0.01** |
| Early stopping 指标 | val_loss | **eval IC**（用 walk-forward eval 集） |
| StochasticDepth | 无 | **加入** (drop_rate=0.1) |
| Stock dropout | 无 | **每截面随机丢弃 20% 股票** |
| Label smoothing | 无 | **分类头 label_smoothing=0.1** |
| Gradient clipping | 1.0 | 保持 |

### 数据增强

- **截面子采样**: 每个训练日期随机选 80% 股票，每 epoch 不同子集
- **时序 jitter**: 序列起始位置 ±2 天随机偏移
- **因子噪声**: 对输入加 Gaussian noise σ=0.05

## 四、标准化改进

### 当前

全局 z-score（训练集 mean/std）。问题：不同日期因子绝对值差异大。

### Phase 2: 截面内 rank 标准化 + 全局 z-score 双通道

```python
# 截面内排名 (0~1)，消除不同日期的绝对值差异
rank_features = cross_section.rank(pct=True)
# 全局 z-score (保留绝对信息)
zscore_features = (cross_section - global_mean) / global_std
# 拼接或加权融合
```

> 备选：只用截面 rank，不保留 z-score。更简洁，但丢失绝对信息。
