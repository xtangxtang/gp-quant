## 三、综合分析

### 结果总结

**Phase 2.1 失败。** Eval IC = -0.0254，不仅不如 Phase 1 (0.0581)，甚至比随机预测还差（负值意味着预测方向与实际相反）。

### 问题 1：截面数据管道 + CrossSectionalEncoder 组合仍然有问题

虽然 Phase 2.1 回退了所有正则化到 Phase 1 水平（仅保留了 stock_dropout=0.05），但结果仍然不如 Phase 1。说明问题不在正则化强度，而在**截面训练范式本身**。

具体表现：
- **Val IC 看起来更好**：Phase 2.1 Val IC=0.1288 vs Phase 1 Val IC=0.906（Phase 1 的 0.906 是过拟合标志，但 Phase 2.1 的 0.1288 也不代表泛化能力）
- **Eval IC 反而更差**：-0.0254 vs 0.0581，差距巨大
- **方向准确率低于 50%**：46.5% 意味着模型预测方向系统性地反了

### 问题 2：CrossSectionalEncoder 是关键问题

Phase 2.1 保留了 CrossSectionalEncoder（265K 额外参数），这是与 Phase 1 唯一的结构性差异。分析：

1. **位置编码无效**：`pos_encoding` shape 为 `(1, 1, d_model)`，广播到所有 stock 后是同一个值。模型无法区分不同 stock 的位置 → 截面 Transformer 等效于一个没有位置信息的 set function
2. **训练/评估前向路径虽然修复了**（都用 forward_cross_sectional），但 CrossSectionalEncoder 学到的表示可能对 unseen 数据泛化能力极差
3. **模型容量翻倍**（575K → 840K）但训练数据量不变（~155K 有效样本/epoch），即使正则化回退，过拟合风险仍然存在

### 问题 3：Val-Eval 关系倒挂

| Phase | Val IC | Eval IC | 关系 |
|-------|--------|---------|------|
| Phase 1 | 0.906 | 0.058 | 过拟合 (Val >> Eval) |
| Phase 2 v1 | 0.075 | 0.055 | 接近 |
| Phase 2 v2 | 0.092 | -0.002 | 反相关 |
| Phase 2.1 | 0.129 | -0.025 | **反相关** |

Phase 2.1 的 Val IC 和 Eval IC 呈**反相关**：Val IC 越高，Eval IC 越低。这说明截面训练优化的目标与真实泛化能力**不一致**。可能的原因：
- 截面内 IC loss 优化的是"同一天内股票的相对排序"，但 eval 看的是跨天的 Pearson 相关
- CrossSectionalEncoder 学到的可能是"某天特定的截面模式"而不是"通用的因子重要性"

### 诊断结论

**截面数据管道本身可能是好的**（IC loss 天然截面内计算，Val IC 计算修正为每日均值），但 **CrossSectionalEncoder 的引入导致了反效果**。

下一步应该做真正的消融：**只保留截面数据管道，关闭 CrossSectionalEncoder**，看数据管道本身的改进是否有价值。

## 四、下一步建议

### Phase 2.2 — 纯截面数据管道（无 CrossSectionalEncoder）

| 项目 | Phase 2.1 | Phase 2.2 建议 |
|------|-----------|---------------|
| CrossSectionalEncoder | 启用 | **关闭**（使用 Phase 1 的 575K 参数模型） |
| 数据管道 | 截面字典 | 保持截面字典 |
| 训练循环 | train_cross_sectional | 截面 batch 但用 Phase 1 forward() |
| Val IC 计算 | 每日 IC 均值 | 保持 |
| 评估前向 | forward_cross_sectional | forward() (Phase 1 路径) |
| 正则化 | Phase 1 水平 | 保持 |

如果 Phase 2.2 Eval IC > Phase 1 → 截面数据管道本身有价值
如果 Phase 2.2 Eval IC ≈ Phase 1 → 截面管道没优势，回到 Phase 1
如果 Phase 2.2 Eval IC < Phase 1 → 截面管道本身就有问题

### 长期：修复 CrossSectionalEncoder

如果要保留截面编码器，需要修复位置编码：
- 使用可学习的 rank-based 位置编码（按股票截面排名分配位置）
- 或使用 permutation-invariant 的 set function（Deep Sets, not Transformer）
