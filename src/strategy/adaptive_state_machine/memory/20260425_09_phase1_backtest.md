> 日期: 2026-04-25
> 状态: 回测完成，4 种状态全部触发
> 结论: 用分位数中位数相对排名替代分类头后，信号正常产出

## 一、代码修改

### 修复 1: torch-to-numpy 转换

模型输出为 torch Tensor，后续 numpy 操作报错。

```python
if hasattr(pred_returns, "detach"):
    pred_returns = pred_returns.detach().cpu().numpy()
    pred_cls = pred_cls.detach().cpu().numpy()
    pred_quantiles = pred_quantiles.detach().cpu().numpy()
```

### 修复 2: 用 q50 相对排名替代分类头 `pred_up_prob`

**问题**：分类头只占 15% loss 权重，`pred_up_prob` 整体偏低，`> 0.5/0.6` 阈值从未触发，导致 0 个 ACCUMULATION 和 0 个 BREAKOUT。

**第一版修复**：用 `q50 > 0` 替代 `pred_up_prob > 0.5`。结果：只产生 2 个 BREAKOUT，因为模型整体偏熊（q50 大多为负）。

**最终修复**：用 q50 的**截面相对排名**替代绝对 `> 0` 阈值。

```python
# 分位数头占 40% loss 权重，比分类头 (15% CE) 可靠
# 但 q50 可能整体偏负（熊市），用相对排名替代绝对 > 0 阈值
q50_pctile = float(np.sum(q50 <= q50_val) / max(len(q50), 1))

if pct > 0.9 and q50_pctile > 0.8:          # BREAKOUT: 前 10% + q50 前 20%
elif pct > 0.7 and q50_pctile > 0.5:        # ACCUMULATION: 前 30% + q50 前 50%
elif pct < 0.1 and q50_pctile < 0.2:        # COLLAPSE: 后 10% + q50 后 20%
elif pct > 0.3:                             # HOLD: 剩余
```

## 三、限制

### 当前回测只是信号扫描器

`backtest()` 方法遍历日期 → 调用 scan → 收集信号，**但不模拟实际持仓和收益计算**。

`backtest_summary.csv` 只记录了信号计数，没有：
- 持仓收益率 / 夏普比率 / 最大回撤
- 前向实际收益（买入后 N 天的真实 P&L）

要得到这些指标，需要从 signals 中提取推荐股票，用真实 OHLCV 数据计算持有期收益。
