# q50 相对排名修复 — 分类头替代方案

> 日期: 2026-04-25
> 状态: 修复完成并验证
> 来源: 更高维度审查建议 — "用分位数中位数 (q50) 替代分类头 pred_up_prob"

## 一、建议原文

> "更干净的做法是直接用分位数中位数（q50）的符号来判断方向：
> 分位数头占 40% 的 loss 权重，是训练最充分的头，比分类头可靠得多。
> 这样也不需要调整任意阈值。"

## 二、问题

原始状态判定依赖分类头的 `pred_up_prob`（15% CE loss）：

```python
if pct > 0.9 and pred_up_prob > 0.6:       # BREAKOUT
elif pct > 0.7 and pred_up_prob > 0.5:     # ACCUMULATION
elif pct < 0.1 and pred_up_prob < 0.4:     # COLLAPSE
```

分类头未校准：`pred_up_prob` 整体偏低，所有股票都 < 0.5，导致 ACCUMULATION 和 BREAKOUT 全部为 0。

## 三、修改代码

### strategy.py — _predict_states_from_model()

**修复 1**：q50 绝对符号替代（v2，不完整）
```python
q50 = pred_quantiles[:, _q_idx[0.5]]
if pct > 0.9 and q50 > 0:      # 结果：只有 2 个 BREAKOUT
```

问题：q50 整体偏负（模型学到熊市 bias），`q50 > 0` 对绝大多数股票不成立。

**修复 2**：q50 截面相对排名替代（v3，最终）
```python
from src.strategy.adaptive_state_machine.attention_learner import QUANTILE_LEVELS
_q_idx = {q: i for i, q in enumerate(QUANTILE_LEVELS)}
q50 = pred_quantiles[:, _q_idx[0.5]]

# 截面相对排名：q50 在当前截面中排第几百分位
q50_pctile = float(np.sum(q50 <= q50_val) / max(len(q50), 1))

if pct > 0.9 and q50_pctile > 0.8:          # BREAKOUT: Sharpe 前 10% + q50 前 20%
elif pct > 0.7 and q50_pctile > 0.5:        # ACCUMULATION: Sharpe 前 30% + q50 前 50%
elif pct < 0.1 and q50_pctile < 0.2:        # COLLAPSE: Sharpe 后 10% + q50 后 20%
elif pct > 0.3:                             # HOLD
```

## 六、总结

建议方向正确：用分位数头替代分类头。但实际落地发现 `q50 > 0` 绝对阈值在熊市环境下不成立（模型整体偏熊），需要加上**截面相对排名**才能产生均衡的信号分布。

相对排名的好处：
- 自动适应牛市/熊市环境（不需要调阈值）
- 与 q50 的训练目标一致（40% loss 权重）
- 不增加任何超参数
