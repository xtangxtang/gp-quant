# 实验 A：因子分组嵌入 — 控制变量重训

> 日期: 2026-04-26
> 前置: `20260426_02_group_embed_diagnosis.md` — 诊断报告指出首次训练存在 3 个混淆变量

## 一、问题

首次分组嵌入训练（v1, 800 股）Eval IC=0.017，远低于 Phase 1 的 0.058。但实验同时改变了 3 个变量：
1. max_stocks: 500 → 800（因子集不同）
2. dropout: 0.1 → 0.2
3. weight_decay: 0.0001 → 0.01

无法归因到分组嵌入本身。

## 二、实验 A 配置（控制变量）

| 参数 | Phase 1 | 实验 A (v2) | 说明 |
|------|---------|-------------|------|
| max_stocks | 500 | **500** | 因子集一致 |
| dropout | 0.1 | **0.1** | 正则化一致 |
| weight_decay | 0.0001 | **0.0001** | 正则化一致 |
| 嵌入方式 | Linear(47→128) | 分组嵌入(11组→fusion→128) | **唯一变量** |

| 指标 | Phase 1 (单层 Linear) | 实验 A (分组嵌入) |
|------|----------------------|------------------|

| 方向准确率 | ~51% | 51.2% |
| Val IC (Spearman) | 0.906 | 0.915 |

## 五、结论

**Eval IC 0.055 vs 0.058 — 基本持平，分组嵌入至少不劣于单层 Linear。**

差异很小（-5%），在随机波动范围内可以说两者等效。Val IC 分组嵌入更高（0.915 vs 0.906），说明排序能力不差，只是前向泛化未显著提升。

### 决策

- 分组嵌入 **可以保留**（引入了因子语义结构，且未损害性能）
- 但 **不能依赖它带来 IC 提升**（从 0.058 → 0.055，无显著改善）
- 后续优化方向应放在其他环节（因子质量、时间窗、特征工程等）
