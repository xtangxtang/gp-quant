## 二、三个核心问题

### 1. 数据量是真正的瓶颈，不是模型架构

- Phase 1：575K 参数，~96K 训练样本 → IC=0.058
- Multi-scale：1.14M 参数，~96K 训练样本 → IC=0.002

参数/样本比从 6:1 膨胀到 12:1，样本量没变，模型容量翻倍。在金融时序这种信噪比极低的场景，**每增加一个参数都需要大量额外数据来约束**。Vision 文档假设"5300 股 × 多年数据"够大，但实际有效独立样本远没那么多（同板块股票高度相关，时间序列自相关）。

**结论**：在当前数据量下，**不要再增加模型容量**。575K 已经是上限。后续精力应投入数据侧，不是模型侧。

### 2. 周线 = 日线 5 日均值，不是新信息

06 文档自己也指出了："周线 = 日线每 5 日均值，信息高度重叠"。这不是"可能原因"，是**确定的原因之一**。日线 60 天序列已经包含了 12 个完整周的信息，再加一个 12 周通道，模型看到的是同一组数据的两种视角，但需要多付出 1 倍参数来处理。

如果要做真正的多尺度，需要的是**不同来源的信息**，比如：
- 周线用**不同于日线的因子**（如只保留周频特有的：周 K 线形态、周成交量分布、周度资金流净额），而不是日线因子的简单均值
- 或者把周线改为 **monthly 级别**，跨度拉长到 24-36 个月，真正引入长周期信号

### 3. 缺少基线复现验证

05 文档清理了代码（回退 Linear、删除 CrossSectionalEncoder），06 文档直接跳到 multi-scale + GPU 训练。但还没有用清理后的代码重跑一遍 Phase 1 确认 IC≈0.058 可复现。05 的代码改动涉及 `__init__`、`_embed()`、`load()`、`extract_weights()`，如果引入了微妙 bug，后续所有实验的基线都是歪的。

## 三、具体建议（按优先级）

### 优先级 1：用清理后代码 + GPU 重跑 Phase 1 基线

```bash
python -m src.strategy.adaptive_state_machine.train_attention \
    --walk-forward --max-stocks 500 --n-epochs 50 \
    --save-path models/phase1_clean_baseline.pt
```

**目标**：确认 Eval IC ≈ 0.058。这是所有后续实验的锚点。

### 优先级 2：市值中性化单独消融

已实现 `--neutralize total_mv`，但只在 multi-scale 里混着跑了，没有单独测试。04 文档发现 `total_mv` 是第三重要因子（权重 2.42），暗示模型可能在吃市值 beta 而非真正的 alpha。

```bash
# 在 Phase 1 架构上，只加市值中性化
python -m src.strategy.adaptive_state_machine.train_attention \
    --walk-forward --max-stocks 500 --neutralize total_mv
```

- 如果中性化后 IC 下降 → 之前的 0.058 里有显著市值偏差，需要重新评估 P&L 的 Sharpe 2.16 是否只是大盘 beta
- 如果中性化后 IC 持平或上升 → 因子里有独立于市值的 alpha，Phase 1 更可信

### 优先级 3：不要继续做 PatchTST

Vision 文档里还剩 PatchTST 没做，但所有经验都指向同一个结论：**在当前数据量和信噪比下，更复杂的架构只会过拟合**。PatchTST 会引入更多参数（patch embedding + 额外的 attention 层），大概率重蹈 multi-scale 的覆辙。

### 优先级 4：如果还想提升 IC，方向在数据侧

| 方向 | 预期投入 | 逻辑 |
|------|---------|------|
| **增加训练年限** | 低 — 用 2022-2025 训练（4 年），eval 2026 | 样本翻倍，参数/样本比回到合理范围 |
| **因子工程** | 中 — 加入基本面因子（PE/PB/ROE/营收增速），来自 `tushare-fina_indicator/` | 47 个因子全是技术面+资金流，缺基本面维度 |
| **行业中性化** | 中 — OLS 回归因子 ~ 行业哑变量 | 比市值中性化更全面 |
| **样本加权** | 低 — 对高波动/高成交量股票加权 | 低质量样本（ST、次新、微小盘）引入噪声 |

## 四、最终建议

**当前 Phase 1（IC=0.058, Sharpe 2.16）就是这套 Transformer 能给出的合理答案。** 后续在这个模型架构上继续加复杂度大概率是负收益。精力分配建议：

- **20%** 验证基线 + 市值中性化消融（确认已有成果的质量）
- **80%** 投入到数据侧改进（更多年份、更好因子、行业中性化），或直接把 Phase 1 模型投入实盘信号生成，在真实交易中积累反馈
