# P&L 分离后的推进计划

**日期**: 2026-04-26
**前置**: `20260426_11_pnl_alpha_beta_separation.md` — Sharpe 1.05，确认有独立 alpha

---

## 一、当前状态评估

Sharpe 1.05 刚过线，信号质量偏脆弱：
- 胜率 47.1%（低于 50%），靠右偏分布（少数大盈利）撑住正收益
- 中位数 -0.58%，大多数交易是亏的
- 最大单笔 +195%（sh688585）是极端异常值，去掉这一笔 Sharpe 会明显下降

后续改进的**容错空间很小**——任何降低 IC 的操作都可能把 Sharpe 打到 1.0 以下。

---

## 二、执行顺序（基本面因子放最后）

| 步骤 | 内容 | 理由 |
|------|------|------|
| **先做 3** | 样本过滤：剔除 ST、次新、微小盘 | 改动最小（只过滤训练数据），不改模型不改因子，风险最低。sh688585 (+195%) 就是科创板次新，去掉这类极端样本后能看到更真实的基线 |
| **再做 4** | 增加训练年限到 2022-2025 | 样本量翻倍但不改架构，配合 3 的清洗效果更好 |
| **然后做 2** | 行业中性化 | 改变因子分布，影响面最大，应该在数据质量（3）和数据量（4）都到位之后再做 |
| **最后做 1** | 基本面因子 | 用户决定放最后 |

每一步都用 **seed=42** 跑，和基线 IC=0.0656 / Sharpe 1.05 直接比。

---

## 三、步骤 3 执行前的风险检查

做样本过滤前，**先统计 613 笔 breakout 信号里有多少是这类股票**：

```python
# 在 P&L 数据中统计:
# - 688 开头（科创板）占比
# - 300 开头（创业板）占比
# - ST 股占比
# - 市值 < 20 亿的占比
# - 上市不足 1 年的次新股占比
```

如果这类股票占比 > 30%，剔除后信号数量会大幅减少，Sharpe 可能因为样本不足而不可靠。

**更关键的是**：如果这类股票贡献了大部分正收益，说明模型的 alpha 来源就是"小盘科创股的高波动"，剔除后 alpha 可能也消失——这本身就是一个重要的信号，需要先确认再决定怎么过滤。
