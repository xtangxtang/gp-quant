# 小盘股 Alpha 分析与推理策略

**日期**: 2026-04-26
**前置**: `20260426_12_signal_composition_analysis.md`

---

## 一、训练数据构成

训练代码 `build_training_data()` 从 `tushare-daily-full/` 随机抽 500 只股票（`np.random.choice`），**无任何过滤**——不排除 ST、不排除次新、不排除微小盘。训练集包含这些股票，但它们是少数（ST ~5%, 次新 ~7%），不是主体。

**模型在小盘股上的优势不是训练偏差。**

---

## 二、为什么模型在小盘股上天然更强

模型学到的核心因子是**波动率+熵+资金流**（coherence_l1, turnover_entropy_l, breakout_range 等），衡量"价格行为的有序性/混乱度变化"。

| 股票类型 | 特点 | 因子信噪比 |
|---------|------|-----------|
| ST/次新/微小盘 | 关注度低、流动性差、信息不对称大 | **高** — 资金集中介入时产生更明显的熵特征变化（从无序到有序的信号更强） |
| 大盘蓝筹 | 大量机构/量化覆盖 | **低** — 同样的因子信号早已被交易消化 |

简单说：模型学到的是"从混乱到有序"的特征，这个特征在小盘股上信噪比天然更高。

---

## 三、数据验证

| 域 | 胜率 | 平均收益 |
|---|---|---|
| < 50 亿 | 51.2% | +3.71% |
| > 100 亿 | 43.2% | +1.51% |
| ST | 53.7% | +8.21% |
| 次新 | 60.5% | +4.48% |

被"排除"的股票贡献了 51% 的总收益，剔除后基线恶化 42%。

---

## 四、两个必须管控的风险

1. **流动性风险**：< 20 亿的股票日成交额可能只有几百万，真实下单时滑点会吞掉大部分收益
2. **生存者偏差**：ST 股虽然平均收益 +8.21%，但单笔亏损可能是退市归零。41 笔样本太少，不能当作稳定 alpha

---

## 五、推理策略建议

不是"只推理小盘"，而是**推理全市场，按市值分层管理仓位**：

```
推理全市场 → composite_score 排序 → 选 Top N
但 Top N 中：
  - < 50 亿的股票：允许满仓位（模型最强区域）
  - 50-200 亿：正常仓位
  - > 200 亿：半仓位或排除
  - 日均成交额 < 2000 万：排除（流动性不够）
  - ST：保留但单只仓位上限 2%（防退市风险）
```

这样既利用了模型在小盘上的优势，又控制了尾部风险。

## 六、对后续计划的影响

| 原计划 | 修正 |
|--------|------|
| 步骤 3：剔除 ST/次新/微小盘 | **跳过** — 这些是 alpha 来源，不是噪声 |
| 步骤 4：增加训练年限 | **继续** — 更多数据让小盘信号估计更稳定 |
| 步骤 2：行业中性化 | **继续** — 在小盘域内做行业中性化仍有意义 |
| 推理策略 | **新增** — 按市值分层仓位管理 |
