# 自适应状态机回测报告

**策略**: `adaptive_state_machine` — 4 Agent 闭环架构
**回测区间**: 2025-01-02 至 2026-03-05
**扫描频率**: 每 20 个交易日一次，共 15 个扫描日期
**报告日期**: 2026-04-24

---

## 一、架构概要

```
Agent 1 (FactorCalculator) → 37 因子计算 (复用 feature_engine.py)
    ↓
Agent 2 (WeightLearner)    → IC 权重更新 + 阈值网格搜索
    ↓
Agent 3 (StateEvaluator)   → 动态阈值状态判定
    ↓
Agent 4 (Validator)        → 历史验证 → 奖励/惩罚 → 反馈 Agent 2
```

### 5 种状态

| 状态 | 含义 | 验证周期 |
|------|------|----------|
| idle | 无信号 | — |
| accumulation | 蓄力阶段 (低熵 + 不可逆性) | 10 日 |
| breakout | 突破信号 (高主特征值 + 放量) | 5 日 |
| hold | 持仓延续 (breakout 入场后) | 10 日 |
| collapse | 崩塌信号 (熵增 + 纯度下降) | 5 日 |

### 因子体系

37 个因子，分 7 类：熵值 (6)、波动率 (4)、特征值 (3)、相干性 (4)、资金流 (6)、价格 (7)、分钟 (7)

## 四、Agent 2 权重更新

- **Config 版本**: 从 v0 递增至 v15，每期稳定更新
- **学习率**: 固定 0.10
- **IC 值**: 极低 (~1e-5 量级)，为 5000+ 股票截面单日快照的典型水平
- **权重更新公式**: `delta_w = lr * IC * (1 + reward)`
- **阈值搜索**: 坐标下降法，20 个阈值参数逐一优化

## 五、问题诊断

### 1. 准确率偏低 (46.8%)
- collapse 占比过高 (80% 已验证预测)，拖累整体
- 当前阈值对 collapse 判定过松，产生大量假阳性
- accumulation 表现相对最好 (50.9%)

### 2. Breakout 信号极度稀缺
- 15 个扫描日期仅 17 次 breakout (平均 1.1 次/期)
- 突破阈值组合要求过严 (需 3/4 条件同时满足)
- 导致 hold 状态几乎不出现

### 3. IC 值过小
- 截面 Spearman 秩相关在 5000+ 样本下单日极低
- 因子权重更新主要靠 Agent 4 奖励信号驱动，IC 贡献微弱
- 建议：改用多日滚动 IC 或 Rank IC 均值

### 4. 状态分布严重不均衡
- 94.2% 的股票处于 idle，有意义信号仅 ~6%
- collapse 占比 (4.7%) 远高于 accumulation (1.1%)
- 策略偏向防守，进攻信号不足

---

## 六、改进方向

1. **放宽 breakout 阈值**: 降低 `dom_eig_breakout`、`vol_impulse_breakout` 的门槛，增加突破信号数量
2. **收紧 collapse 阈值**: 提高 `collapse_need_n` 从 3 到 4，减少假阳性崩塌信号
3. **多日滚动 IC**: Agent 2 使用过去 N 天的 IC 均值而非单日 IC，提高权重更新稳定性
4. **增加因子**: 引入基本面因子 (PE、PB、ROE) 和动量因子，弥补纯技术因子的不足
5. **状态生命周期约束**: 对 accumulation 加入持续时间要求 (如连续 3 天出现才算有效)
6. **动态学习率**: 市场波动大时降低 lr，趋势明确时提高 lr

## 八、结论

4 Agent 闭环架构已成功跑通全量回测 (5,300 只股票 × 15 期)，配置动态更新机制正常工作 (v0→v15)。

但当前预测准确率 (46.8%) 与随机猜测相当，核心问题在于：
- 因子判别力不足 (IC 极低)
- 状态阈值过于保守 (collapse 泛滥)
- 进攻性信号稀缺 (breakout 几乎不触发)

架构本身没有问题，需要在因子选择和阈值优化上做进一步迭代。
