# Bull Hunter v4 — 策略完整总结 (2026-04-20)

> 7 Agent 流水线系统，选股+买卖全闭环，7253 行代码
> 2025 全年回测: **+32.7% 总收益率**，盈亏比 2.32

---

## 一、策略定位

**目标**: 从 A 股全市场 ~5000 只股票中，找出未来 6 个月有可能涨 200% 的"大牛股"，并通过自动化组合管理持续买卖。

**方法**: 7 个 Agent 组成两条管线:

```
选股管线: Agent 1 (因子快照) → Agent 2 (训练) → Agent 3 (预测) → Agent 4 (监控)
买卖管线: Agent 5 (组合管理) ← Agent 6 (退出信号) ← Agent 7 (监督调优, 反馈回路)

完整闭环: Agent 1 → Agent 3 → Agent 6 → Agent 5 → Agent 7 → (反馈 Agent 5/6)
```

---

## 二、7 个 Agent 各司其职

| Agent | 名称 | 职责 | 核心技术 |
|-------|------|------|---------|
| **1** | 因子快照 | 每日生成全市场 ~5000 只股票的 32 维因子截面 | 熵指标 + 波动率 + 资金流 + 密度矩阵 |
| **2** | 模型训练 | 回看 1 年数据训练 LightGBM 分类器 | 200% (主模型, 120日) + 100% (辅助, 40日) |
| **3** | 预测评级 | 全市场概率排序，输出 Top 5 A 级候选 | 概率阈值过滤 + 去重 |
| **4** | 健康监控 | 漏网之鱼检测 + 模型诊断 + 可触发 Agent 2 重训 | 双回路反馈 |
| **5** | 组合管理 | 先卖后买：止损/trailing stop/Agent 6 信号 → 买入候选 | 10 仓位 × 10 万/仓 |
| **6** | 退出信号 | 14 个退出因子加权评分 (动量衰减/熵无序/资金流反转等) | 规则引擎 + LightGBM (可训练) |
| **7** | 监督调优 | 监控买入质量/卖出时机/信号有效性，下发调参指令给 Agent 5/6 | 自适应阈值调整 |

### Agent 5 卖出决策优先级 (先卖后买)

1. **止损** (`stop_loss`): 持仓跌幅 > 15% → 强制卖出
2. **trailing stop**: 最高涨幅 > 50% 后从高点回撤 > 20% → 卖出
3. **Agent 6 信号** (`exit_signal`): sell_weight > 0.6 阈值 → 卖出
4. 满足最小持有天数 (5 天) 后才执行止损/止盈

### Agent 6 的 14 个退出因子

| 类别 | 因子 | 权重 | 含义 |
|------|------|------|------|
| 动量衰减 | `momentum_decay` | 0.12 | 20d 动量由正转负 |
| 动量加速度 | `momentum_accel` | 0.10 | 10d 动量减速 |
| 熵无序化 | `entropy_disorder` | 0.10 | 短期熵高于中期 |
| 资金流出加速 | `mf_outflow_accel` | 0.10 | 主力净流出加速 |
| 波动率异常 | `vol_expansion` | 0.08 | 短期/长期波动率比 > 2.0 |
| 路径不可逆崩塌 | `irrev_collapse` | 0.08 | 不可逆度从高位下降 |
| 跌破 MA60 | `below_ma60` | 0.08 | 跌破 60 日均线 |
| 从最高点回撤 | `drawdown_from_peak` | 0.08 | 回撤幅度 / 30% |
| 连续净流出 | `mf_streak_negative` | 0.06 | 连续流出天数 / 5 |

## 六、架构特点

1. **模型轮换无前瞻偏差**: `get_model_for_date()` 确保回测中每天只用当时已有的模型
2. **预加载缓存**: ~10000 个 CSV 一次性读入内存 (`preload_factor_cache`)，每日处理从 ~90s 降至 ~8s
3. **向量化预测**: Agent 3 的特征矩阵构造从 Python 双循环改为 pandas/numpy，~25s → <1s
4. **双反馈回路**:
   - Agent 4 反馈选股质量 → Agent 2 重训
   - Agent 7 反馈买卖质量 → Agent 5/6 调参

## 七、已知问题

| 问题 | 严重度 | 说明 |
|------|--------|------|
| Agent 6 退出信号从未触发卖出 | ⚠️ 高 | 14 个退出因子加权分始终 < 0.6 阈值，卖出全靠 Agent 5 的止损/trailing stop |
| Agent 6 LightGBM 模型未训练 | ⚠️ 中 | 全年 16 笔卖出不够训练门槛，已实现训练代码 (`train_exit_model`) 待充足样本后验证 |
| 期末满仓锁死 | ⚠️ 中 | 10 只仓位满后新候选无法买入，部分持仓已从高点大幅回撤 (如罗牛山从 +31.3% 回到 +5.8%) |
| Agent 7 调优幅度有限 | ℹ️ 低 | 单次最大调 0.05，多轮累积后 sell_threshold 从 0.60 升到 0.65，但 Agent 6 分数远低于此 |
| 0407 集中止损 | ℹ️ 低 | 6 笔同日止损 (系统性下跌)，需考虑大盘过滤或分批止损 |

## 八、下一步优化方向

1. **Agent 6 退出因子调优**: 降低阈值 (如 0.4) 或增大高贡献因子权重，让 exit_signal 真正参与卖出决策
2. **大盘过滤**: 在 Agent 5 买入前加入 market_trend 判断，市场极弱时暂停买入或降低仓位
3. **Agent 2 定期重训**: 使用 `--retrain_interval 60` 每 60 天自动重训，保持模型时效性
4. **持仓周转优化**: 引入最大持有天数强制卖出 (如 120 天)，释放资金给新候选
5. **Agent 6 模型训练验证**: 积累更多交易样本后验证 LightGBM 退出模型效果
