# Entropy Hold Judgement

## 描述

一句话概括：用熵、相对有序度和局部动力记忆来判断一只股票自某个起始时间起是否还能一致持有。

这套策略不是做“选股入场”，而是做“持有判定”。输入一只股票和一个起始日期后，策略会检查从起始日至当前扫描日之间，是否出现了持续性的高熵乱序、相对有序结构破坏以及动力记忆坍缩。

核心思想：

- 不用均线金叉或 `close > ma_20`
- 继续沿用熵和非线性动力学框架
- 把“退出”定义成：高熵乱序持续 + 记忆坍缩持续

主要看三类量：

- `entropy_reserve`：系统还剩多少有序余量
- `memory_reserve`：局部动力系统的记忆是否还在
- `disorder_pressure`：高熵乱序是否在持续积累

## 主要参数

- `symbol_or_name`：股票代码或股票名称，例如 `sh603507`、`603507.SH` 或 `振江股份`
- `start_date`：起始持有日期，格式 `yyyymmdd`
- `scan_date`：评估日期，默认自动推断最新交易日
- `data_dir`：日线 CSV 目录
- `basic_path`：`tushare_stock_basic.csv` 路径，用于名称和代码映射
- `lookback_years`：特征回看年数，默认 5
- `exit_persist_days`：连续多少天满足退出条件才判定无法继续持有，默认 3

## 输出

策略会输出：

- 当前是否建议继续持有
- 首次失效日期（如果存在）
- 当前持有分数、乱序压力、熵分位、记忆强度
- 一份完整的日度诊断时间序列 CSV
